## What is foreman-builder?

foreman-builder is a command-line tool for Mac developers who need to run [Foreman](https://theforeman.org/) — the open-source infrastructure management platform — in a container-based local development environment.

Rather than spending hours manually configuring a virtual machine, foreman-builder provisions a full Foreman/Katello development stack inside an [OrbStack](https://orbstack.dev/) Linux container with a single command. The result feels as close to running locally as possible: ports opened in the container are accessible via `localhost`, and you can SSH directly into the environment.

## How it works

foreman-builder uses [cloud-init](https://cloud-init.io/) to provision Rocky Linux 9 containers via OrbStack. When you run `foreman-builder create`, it:

1. Reads your local username and any optional `config.yml` packages
2. Generates a cloud-init configuration from a built-in template
3. Creates a Rocky Linux 9 container in OrbStack with that config
4. The container automatically installs all Foreman dependencies and runs the `katello_devel` Ansible playbook
5. Records the container name in `~/.foreman-builder/containers` for tracking

Once the container is running, OrbStack makes all ports accessible via `localhost`, so your Foreman instance is immediately reachable in your browser.

## Requirements

Before using foreman-builder, you need:

- **macOS with Apple Silicon** (arm64) — Intel Macs are not currently supported
- **Go 1.25.5+** — to install the binary via `go install`
- **OrbStack** — must be installed and running

See the [Requirements](/requirements) page for installation links and details.
